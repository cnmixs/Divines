# Superpowers 开发流程技能集

基于 [obra/superpowers](https://github.com/obra/superpowers) v5.1.0，提供完整的开发流程最佳实践。

## 技能 1：头脑风暴 (Brainstorming)

**必须在任何创造性工作之前使用** — 创建功能、构建组件、添加功能或修改行为前。

### 流程
1. **探索项目上下文** — 检查文件、文档、最近提交
2. **提供可视化辅助**（如果涉及视觉问题）
3. **逐一提问澄清** — 理解目的/约束/成功标准
4. **提出 2-3 种方案** — 含权衡分析和推荐
5. **展示设计方案** — 按复杂度分段，每段获得用户确认
6. **写入设计文档** — 保存到 `docs/superpowers/specs/YYYY-MM-DD-<topic>-design.md`

### 硬性规则
> 在展示设计方案并获得用户批准之前，**不要**调用任何实现技能、编写代码、搭建项目或采取任何实现行动。这适用于**所有**项目，无论看似多简单。

## 技能 2：执行计划 (Executing Plans)

当有书面实现计划时使用。

### 流程
1. **加载并审查计划** — 批判性审查，识别问题
2. **执行任务** — 逐一标记 in_progress → completed，按步骤执行验证
3. **完成开发** — 使用 finishing-a-development-branch 技能收尾

### 停止条件
- 遇到阻塞（缺少依赖、测试失败、指令不清）
- 计划有严重缺陷
- 不理解指令
- 验证反复失败

## 技能 3：并行 Agent 调度 (Dispatching Parallel Agents)

当面对 2+ 个独立任务时可以并行处理。

### 使用时机
- 3+ 个测试文件失败且根因不同
- 多个子系统独立损坏
- 每个问题可独立理解

### 模式
1. **识别独立领域** — 按问题域分组
2. **创建聚焦的 Agent 任务** — 每个 Agent 有明确范围、目标和约束
3. **并行调度** — 同时运行多个 Agent
4. **审查和集成** — 验证修复不冲突，运行完整测试

### Agent Prompt 结构
- **聚焦** — 一个清晰的问题域
- **自包含** — 所有理解问题所需的上下文
- **明确输出** — Agent 应返回什么

## 技能 4：Git 工作树 (Using Git Worktrees)

### 核心原则
- 每个功能/修复在独立的工作树中进行
- 主分支保持干净，只用于拉取和合并
- 工作树路径：`../<repo>-<branch>`

## 技能 5：完成开发分支 (Finishing a Development Branch)

### 流程
1. 运行完整测试套件验证
2. 检查代码质量和 lint
3. 展示选项给用户（提交、PR、放弃等）
4. 执行用户选择

## 反模式

- **"这太简单了不需要设计"** — 每个项目都需要设计，简单的项目更需要明确假设
- **跳过验证** — 永远不要跳过计划中的验证步骤
- **猜测而非询问** — 遇到阻塞时停止并询问，不要猜测
