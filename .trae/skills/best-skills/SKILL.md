# 最佳技能合集

基于 [xstongxue/best-skills](https://github.com/xstongxue/best-skills)，中文开发流程技能集。

## 技能 1：开发流程五步法 (dev-workflow)

**需求理解 → 方案设计 → 代码实现 → 代码审查 → Bug 修复**

### 流程串联
| 步骤 | 触发关键词 | 说明 |
|------|------|------|
| 需求理解 | 需求分析、理解需求、帮我梳理 | 从描述中提取功能想法、约束条件 |
| 方案设计 | 方案设计、技术设计、架构设计 | 确认需求后设计技术方案 |
| 代码实现 | 代码实现、开始写代码、帮我实现 | 确认方案后编写代码 |
| 代码审查 | 代码审查、Review、检查代码 | 明确审查范围，提供结构化反馈 |
| Bug 修复 | bug、报错、崩溃、异常 | 确认错误信息、复现步骤、环境 |

### 自动落盘规则
- **需求理解** → 写入 `docs/需求理解.md`
- **方案设计** → 写入 `docs/方案设计.md`
- **代码审查** → 写入 `docs/代码审查.md`
- 多模块项目优先使用 `docs/<module>/` 分目录
- 每次写入前添加时间标题：`## YYYY-MM-DD HH:mm`
- 不覆盖历史内容，追加写入

## 技能 2：代码生成图表 (codegen-diagram)

生成 UML 图、流程图、架构图等可视化图表。

## 技能 3：代码生成文档 (codegen-doc)

基于当前项目/代码仓库生成文档，支持四种类型：

| 类型 | 说明 |
|------|------|
| 论文章节 | 系统设计、总体设计、详细设计 |
| 项目梳理 | 项目文档结构、按格式梳理 |
| 重点问题 | 技术难点、待解决问题、项目风险 |
| 简历项目描述 | 项目经历、按简历格式 |

### 通用原则
- **不编造**：未在仓库中出现的内容不写入
- **有据可依**：从代码、注释、README、文档中抽取
- **格式遵从**：严格按用户提供的格式组织输出

## 技能 4：专利撰写 (patent-writer)

辅助撰写专利申请文档，包括技术方案描述、权利要求书等。

## 技能 5：代码审查 (code-review)

结构化代码审查流程：
- 代码质量检查
- 安全性审查
- 性能分析
- 最佳实践建议

## 发展检查清单

进行任何开发任务时：
- [ ] 确认当前处于哪个流程阶段
- [ ] 检查上游步骤是否完成
- [ ] 按流程步骤执行
- [ ] 关键步骤自动落盘文档
